# Reel ‚Äî rank your movies and series

A personal pairwise ranker for everything in `data/imdb_ratings.csv`. Modeled
on `food-ranker`: Next.js 14 + Supabase + magic-link auth + per-user
comparisons + global Elo. The whole point is **resolving the ties**: a
1‚Äì10 IMDb rating leaves you with hundreds of titles all sitting at "8/10",
and this app turns that into a full order via head-to-head choices.

> Lives at `~/website/movie-ranker/`, alongside `food-ranker/` and
> `detour-finder/`. Standalone Next.js app, **not** part of the R Markdown
> GitHub Pages build. Independently deployed (Vercel / Netlify / Fly).

## Stack

- Next.js 14 (App Router) + TypeScript
- Tailwind CSS (dark-mode-first)
- Supabase (Postgres + Auth)

## Setup

### 1. Install

```bash
cd ~/website/movie-ranker
npm install
```

### 2. Build the seed from your IMDb CSV

```bash
npm run build-seed
```

Reads `~/website/data/imdb_ratings.csv` and writes
`supabase/seed.sql`. Re-run any time you re-export from IMDb ‚Äî the SQL is
idempotent (`on conflict (tconst) do update`), so existing rows keep their
global Elo while metadata refreshes.

### 3. Create a Supabase project

In the SQL Editor, run in order:

1. `supabase/schema.sql` ‚Äî creates `titles`, `comparisons`, indexes, RLS.
2. `supabase/seed.sql` ‚Äî inserts every title from the CSV.

### 4. Configure environment

```bash
cp .env.local.example .env.local
```

Fill in the project URL and anon key (Settings ‚Üí API).

### 5. Run

```bash
npm run dev
```

Open <http://localhost:3000>. Sign in with your email, then start the
comparison loop on `/compare`.

## How the rating works

### Per-bucket starting Elo

Every title is seeded with `800 + your_rating √ó 50`:

| `your_rating` | starting Elo |
|---|---|
| 1 | 850 |
| 5 | 1050 |
| 8 | 1200 |
| 9 | 1250 |
| 10 | 1300 |

The 50-point gap between buckets means a typical Elo swing (~16 points at
K = 32) can&apos;t flip a 9 below an 8 just from a few unlucky comparisons.
Inside a bucket every title starts equal ‚Äî you&apos;re building the order
yourself.

### Elo update

Standard Elo with K = 32, identical to `food-ranker`:

```ts
const expected = 1 / (1 + 10 ** ((loser - winner) / 400));
newWinner = winner + 32 * (1 - expected);
newLoser  = loser  + 32 * (0 - (1 - expected));
```

### Personal vs global

- **Personal** (default): comparisons are scoped to your user. The
  leaderboard re-derives your personal Elo by replaying *your* comparisons
  in chronological order, starting from the per-bucket anchor. Voting on
  `/compare?scope=personal` updates only your personal ranking.
- **Global**: the shared `titles.rating` column. Voting on
  `/compare?scope=global` writes to it for everyone. Both viewers see the
  same global numbers.

## Matching logic

`src/lib/matching.ts` is built around tie-resolution:

1. Pool is filtered to a single kind (movies vs series vs ...). Movies +
   TV movies group together; series + mini-series group together.
2. Group the pool by `your_rating`. Pick a star bucket weighted by `n¬≤`
   so the 8/10 bucket of 150 isn&apos;t starved by the 10/10 bucket of 18.
3. Inside the bucket, pick uniformly at random for the first title, then
   prefer a partner whose current Elo is within ¬±100 of the first&apos;s.
4. Avoid reproducing the immediately previous matchup.

If a star bucket has only one title, the picker falls back to neighbouring
buckets so the user always sees a question.

## Re-importing your IMDb ratings

Drop a fresh export at `~/website/data/imdb_ratings.csv`, then:

```bash
npm run build-seed
```

Apply `supabase/seed.sql` in the Supabase SQL Editor. New titles are
inserted; existing titles get their metadata + `your_rating` refreshed but
keep their Elo and comparison history. Removed titles stay in the DB
(comparisons reference them); delete them manually if you want them gone.
